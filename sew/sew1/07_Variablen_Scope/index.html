Sichtbarkeit von Variablen

Softwareentwicklung 1

Was bedeutet "Sichtbarkeit"?

Die Sichtbarkeit einer Variable ist der Bereich des Programmtexts, in dem man die Variable verwenden darf. An allen anderen Stellen im Programm kennt der Compiler/Interpreter den Variablennamen nicht, ist sie nicht sichtbar.

  • Name und Datentyp einer Variable legt man dadurch fest, was man bei der Deklaration hinschreibt.
  • Die Sichtbarkeit bestimmt man (vorläufig nur) dadurch, wo man die Variablendeklaration platziert.

Block

  • Ein Block startet mit einer geschwungenen Klammern { und endet mit einer geschwungenen Klammer }.
  • Eine Variable ist innerhalb des Blocks sichtbar, in der sie deklariert wird.

{
	//innerhalb des Blocks
}

Beispiel Blöcke


			void setup() {

			}

			void draw() {
			
			}

Beispiel 1


	void setup() {
		int groesseAugen = 10;
	}

	void draw() {
		ellipse(10,20,groesseAugen, groesseAugen);
	}

Die Variable groesseAugen ist in draw() nicht sichtbar und kann nicht verwendet werden.

Beispiel 2


void setup() {
	int groesseAugen = 10;
	ellipse(10,20,groesseAugen, groesseAugen);
}

void draw() {
	
}

Die Variable groesseAugen wird nur innerhalb des Blocks verwendet in der sie deklariert wurde.

Globale Variablen

  • Variablen, die außerhalb jedes Blocks stehen, werden als globale Variable bezeichnet.
  • Sie sind im gesamten Programm sichtbar.
  • Wichtig: Globale Variablen sollten so wenig wie möglich eingesetzt werden, da (größere) Programme dadurch sehr schwer zu verstehen sind!
    • Werte können über Parameter von einer Funktion zu einer anderen weiter gegeben werden.
    • In Processing sind sie aber oft notwendig, da draw() keine Parameter hat.

Beispiel lokale & globale Variable


int groesseAugen = 10;


void setup() {
	int groesseMund; 
	groesseMund = groesseAugen + 20;
}

void draw() {
	ellipse(10,20,groesseAugen, groesseAugen);
}

Gleiche Variablennamen


	int y=150;
	int x=200;
	
	void zeichneAugen(int abstand, int groesse){
	    int x = 100;
	    ellipse(x,150, groesse,groesse); //x=100
	    ellipse(x+abstand,150,groesse,groesse);
	}

Hat eine lokale Variable den gleichen Namen wie eine globale Variable, gilt die lokale Variable! Vermeiden!!

Programmierstil - globale/lokale Variablen


	int x=200;
	int abstand=200;
	int groesse=200;
		
	void zeichneAugen(){
	    ellipse(x,150, groesse,groesse); 
	    ellipse(x+abstand,150,groesse,groesse);
	}
	
	void setup(){
		zeichneAugen(); 
	}

Die Variablen abstand und groesse sind global definiert, obwohl sie nur in der Funktion zeichneAugen() verwendet werden.

Verbesserungsvorschläge?

Programmierstil - globale/lokale Variablen v2


int x=200;
		
void zeichneAugen(){
	int abstand=200;
	int groesse=200;

	ellipse(x,150, groesse,groesse); 
	ellipse(x+abstand,150,groesse,groesse);
}

void setup(){
	zeichneAugen(); 
}

Jetzt sind alle Variablen nur in dem Block definiert, in dem sie auch verwendet werden.

Wie könnte man den Code noch besser machen?

Programmierstil - globale/lokale Variablen v3


void zeichneAugen(int x, int abstand, int groesse){
	ellipse(x,150, groesse,groesse); 
	ellipse(x+abstand,150,groesse,groesse);
}
	
void setup(){
	int x=200;
	int abstand=200;
	int groesse=200;
	
	zeichneAugen(x, abstand, groesse); 
}

Die Funktion zeichneAugen()verwendet Parameter um die Größe und den Abstand festzulegen.

Dadurch kann die Funktion viel flexibler eingesetzt werden.

Zusammenfassung

  • Variablen können immer nur innerhalb des Blocks { ... }, in dem sie deklariert wurden, verwendet werden.
  • Deklariere Variablen immer so, dass sie überall sichtbar sind, wo sie verwendet werden, aber sonst nicht sichtbar sind.
  • Verwende, wenn möglich, Parameter um Werte von einer Funktion an eine andere zu übergeben (statt globaler Variablen).
