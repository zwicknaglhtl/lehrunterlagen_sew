Strings

  • Als String bezeichnet man eine endliche Zeichenkette (vereinfacht ausgedrückt einen "Text").
  • String-Literale schreibt man (in Java) zwischen doppelte Hochkomma "".
  • String-Literale können in String-Objekten vom Datentyp String gespeichert werden. 

String subject = "Softwareentwicklung";
String saying = "Be the change you want to see in the world.";

Primitive vs komplexe Datentypen

  • Primitive Datentypen speichern nur einen einzelnen Wert.
    • kennen wir bereits: int, float, boolean
    • werden wir noch kennen lernen: char, byte, short, long, double
  • Komplexe Datentypen beinhalten zusätzliche Funktionalität und können evt. mehrere Werte speichern.
    • werden wir heuer kennen lernen: String, Array
    • viel mehr dazu im 2. Jahrgang

String "Funktionalitäten"

  • Zusätzlich zum Speichern einer Zeichenkette besitzen Strings Funktionalitäten (z.B. um den 3. Buchstaben zu bekommen.)
  • Dazu besitzt jeder String Funktionen, die man bei komplexen Datentypen allgemein "Methoden" nennt.
  • Hängt man an den String / die Stringvariable einen "." (Punkt) kann man die Methoden eines Strings aufrufen.

String-Methoden

  • Java bietet viele Methoden zum Bearbeiten und Untersuchen von Strings.
  • Mit equals() haben wir bereits eine Methode kennen gelernt.
  • Die Dokumentation der Java API bietet eine komplette Übersicht über alle verfügbaren Methoden.

String s = "Act responsible";
int lng = s.length();
println(s.toUpperCase());

Strings & Index

  • „Index“ bedeutet bei einem String die Zeichenposition.
  • Das erste Zeichen hat den Index 0 und das letzte den Index length()-1.

Stringmethoden

  • Folgende Abkürzungen werden in den folgenden Tabellen benützt:
    • index: ein Index (Datentyp int)
    • text1, text2: Text (Datentyp String)
    • ch: ein Zeichen (Datentyp char)

Strings untersuchen

text1.length() Liefert die Anzahl der Zeichen von text1.
text1.charAt(index) Liefert das Zeichen, das sich bei text1 an der Position index befindet.
text1.equals(text2) Liefert true, wenn der text1 gleich text2 ist.
text1.equalsIgnoreCase(text2) Wie equals(), zwischen Klein- und Großschreibung wird nicht unterschieden.
text1.startsWith(text2) Liefert true, wenn text1 mit text2 beginnt.
text1.endsWith(text2) Liefert true, wenn text1 mit text2 endet.
text1.contains(text2) Liefert true, wenn text2 in text1 enthalten ist.
text1.indexOf(ch)
text1.indexOf(text2) 		Liefert den Index des ersten Vorkommens von ch bzw. text2 in text1, oder -1, wenn nicht gefunden.
text1.indexOf(ch, index)
text1.indexOf(text2,index) 		Wie oben, die Suche beginnt aber erst an der Position index.

Beispiele


					String sentence = "Wir lernen Strings!";

					int len = sentence.length();    //19 (weil der String 19 Zeichen hat)
			
					char c = sentence.charAt(5);    //'e' (das Zeichen an der Indexpos. 5)
			
					int i = sentence.indexOf("r");  //2 (der Index des ersten "r")
					i = sentence.indexOf("r",3);    //6 (der Index des ersten "r" ab Pos. 3)
					i = sentence.indexOf("nen");	//7 (der (Start-)Index des ersten "nen")

Strings bearbeiten

Beachte: Die folgenden Methoden ändern nicht den String selbst, sondern liefern den geänderten String als Ergebnis! Man benötigt daher (meist) eine Zuweisung.

text1.toLowerCase() Wandelt text1 in Kleinbuchstaben um.
text1.toUpperCase() Wandelt text1 in Großbuchstaben um.
text1.trim() Entfernt Leerzeichen vom Beginn und vom Ende des Strings text1.
text1.substring(index) Liefert den Teil von text1 von Position index bis zum Ende.
text1.substring(index, ende) Liefert den Teil von text1 von Position index bis ende-1.
text1.replace(ch1, ch2)
text1.replace(text2, text3) 		Ersetzt in text1 alle Vorkommnisse von ch1 durch ch2 (bzw. text2 durch text3).
text1.split(text2) Zerteilt text1 überall, wo text2 vorkommt, und liefert die Teile in einem String[].

Beispiele


					String sentence = "Wir verändern Strings!";

					System.out.println(sentence.toUpperCase());
			
					String firstWord= sentence.substring(0,3); //Wir
			
					String message = sentence.replace(" ", "-"); //Wir-verändern-Strings!

String vs. Zahlen

"1" '1' 1
String char int
  • Strings und Zahlen (int, long,...) sind verschiedene Datentypen, die sich verschieden verhalten - auch wenn sie am Bildschirm gleich erscheinen.
  • Auch wenn ein String wie eine Zahl aussieht, so kann man ihn dennoch nicht ohne weiteres wie eine Zahl verwenden.
  • Einen String, der so aussieht wie eine Zahlenkonstante, kann man jedoch in eine Zahl des passenden Datentyps umwandeln.

Strings in Zahlen umwandeln

Byte.parseByte(text) Wandelt text in eine byte-Zahl um und liefert diese als Ergebnis.
Integer.parseInt(text) Wandelt text in eine int-Zahl um und liefert diese als Ergebnis.
Long.parseLong(text) Wandelt text in eine long-Zahl um und liefert diese als Ergebnis.
Float.parseFloat(text) Wandelt text in eine float-Zahl um und liefert diese als Ergebnis.
Double.parseDouble(text) Wandelt text in eine double-Zahl um und liefert diese als Ergebnis.

Beachte: Entspricht text nicht einer Zahl des gewünschten Datentyps, so stürzt das Programm (derzeit noch) ab.

Beispiele


					String s = "2";
					int n = 3;
					System.out.println(s+n); //Ausgabe: 23
			
					int n2 = Integer.parseInt(s);
					System.out.println(n + n2); //Ausgabe: 5
			
					float z = Float.parseFloat("2.3");
					
					int n3 = Integer.parseInt("2.3");   //NumberFormatException - Programm stürzt ab

String und char


			System.out.println('a' + 'c');	//Ausgabe: 196
  • Ein char ist eigentlich ein Zahlenwert (der Unicode des Zeichens).
  • Der + Operator addiert also den Zahlenwert von 'a' (97) und 'c' (99).
  • Mindestens einer der Operanden muss ein String sein, damit der + Operator Strings verkettet.
    • Durch Verkettung mit einem Leerstring "" kann so ein char einfach in einen String umgewandelt werden.

				System.out.println(""+'a' + 'c'); //Ausgabe ac
				
				('a'+"").toLowerCase(); //'a'+"" wandelt den char 'a' in den String "a" um

Zeichenbereiche prüfen - 1. Versuch

  • Da die Zahlen 0-9 in der Unicodetabelle hintereinander stehen, kann einfach überprüft werden ob ein gewisser char eine Ziffer ist. 
    
					if(c >= '0' && c <= '9'){ ... }
  • Auf ähnliche Weise könnte auf Buchstaben überprüft werden. 
    
						if((c >= 'a' && c <= 'z') || (c >= 'A' && c <= 'Z')){ ... }
  • Diese Überprüfung kann aber zu einigen Problemen führen:
    • Umlaute (ö,ä,...) werden nicht als Buchstaben erkannt und müssten extra überprüft werden.
    • Buchstaben mit Accents (z.B. é, â) oder ähnlichen Erweiterungen werden nicht erkannt.
    • In der Unicodetabelle gibt es auch Ziffern, die so nicht erkannt werden (z.B. Vollbreite Ziffern).
    • Deshalb stellt die Javaklasse Character Methoden zum Testen von Zeichen bereit.

Klasse Character

Character.isDigit(ch) Liefert true wenn ch eine Ziffer ist.
Character.isLetter(ch) Liefert true wenn ch ein Buchstabe ist. Achtung: auch für nicht lateinische Buchstaben wird true geliefert (z.B. sonstige Buchstaben)
Character.isLowercase(ch) Liefert true wenn ch ein Kleinbuchstabe ist.
Character.isUpperCase(ch) Liefert true wenn ch ein Großbuchstabe ist.

Die Klasse Character besitzt noch etliche weiterer solcher "Testmethoden".
