Was sind Bitoperationen?

Anstatt auf der Ebene von primitiven Datentypen oder Objekten zu arbeiten, arbeiten wir mit Bits.

1000101110100101

Einsatzgebiet

  • In der hardwarenahen Programmierung wird oft mit einzelnen Bits gearbeitet.
    • Aktivieren/Deaktivieren einer einzigen Datenleitung -> 1 Bit
    • Fasst mehrere solcher Leitungen (Ports) in einem Datenwort zusammen
  • "Flags" (einzelne Bits) in einer Variable zu speichern statt in vielen boolean Variablen.

Wiederholung

Zweierkomplement und Bitoperationen sollten aus GINF bereits bekannt sein!

Darstellung ganzer Zahlen

int und long in Java

In Java hat ein int 32 Bits.

0000 0000 0000 0000 0000 0000 0000 0000

Ein long hat 64 Bits.


 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000 0000

Zweierkomplement – positive Zahlen

  • Um negative und positve Zahlen einfach speichern zu können, werden die Zahlen als Zweierkomplement kodiert.
  • Das Bit an der höchsten Stelle (ganz links) ist bei negativen Zahlen 1, bei positiven Zahlen 0.
  • Bei n Stellen ist die größte positive darstellbare Zahl daher 2(n-1)−1
    • 1 Stelle für das Vorzeichen, bleiben n-1 Stellen für die Darstellung der Zahl selbst
    • -1 weil wir 0 auch darstellen müssen

Zweierkomplement – negative Zahlen

Negative Zahlen haben an der höchsten Stelle eine 1. 

Beispiel (32-bit):
1101 1110 0111 0000 0111 0000 0100 0110

Bei n Stellen ist die kleinste (negative) darstellbare Zahl daher −2(n-1)

Zweierkomplement

  • Um von einer positiven Zahl die Darstellung der negativen Zahl zu erhalten werden...
    1. alle Bits invertiert (0 wird 1, 1 wird 0)
    2. +1 addiert

0000 0000 0000 0000 0000 0000 0000 0100 -> 4 als Binärzahl
1111 1111 1111 1111 1111 1111 1111 1011 -> invertiert
1111 1111 1111 1111 1111 1111 1111 1100 -> +1 => −4 im Zweierkomplement

Hexadezimalsystem

Zur einfacheren Darstellung von Binärzahlen wird oft das Hexadezimalsystem verwendet.


            1111 0000 1100 1001 1111 1111 1111 0110
             F    0    C    9    F    F    F    6

             -> 0xF0C9FFF6

Bit-Operationen Übersicht

  • & Bitweises UND
  • | Bitweises ODER
  • ^ Bitweises XOR
  • ~ Bitweises INVERTIEREN
  • <<, >>, >>> Shifts

& Bitweises UND

Bitweises UND (&) → Beide Bits müssen 1 sein, damit das Ergebnis 1 ist. 


  1100 0100
& 0010 0101
= 0000 0100

| Bitweises ODER

Bitweises ODER (|) → Mindestens ein Bit muss 1 sein.


  1100 0100
| 0010 0101
= 1110 0101

^ Bitweises Exklusives Oder (XOR)

Bitweises Exklusives Oder (^, XOR) → Genau ein Bit darf 1 sein. 


  1100 0100
^ 0010 0101
= 1110 0001

~ Bitweises Invertieren

Bitweises invertieren (~) → Jedes Bit wird umgedreht. 

~1100 0100 = 0011 1011

<< Nach links schieben

Bits werden beliebig viele Stellen nach links geschoben, rechts werden Nullen aufgefüllt. 

1100 0100 << 2 = 0001 0000

>> Arithmetischer Rechtsshift

Bits werden beliebig viele Stellen nach rechts geschoben, links werden die Vorzeichenbits aufgefüllt.


            1100 0100 >> 2 = 1111 0001  
            0100 0100 >> 2 = 0001 0001

>>> Logischer Rechtsshift

Bits werden beliebig viele Stellen nach rechts geschoben, links werden Nullen aufgefüllt.


            1100 0100 >>> 2 = 0011 0001
            0100 0100 >>> 2 = 0001 0001

Zusammenfassung

  • Bitoperationen sind effizient für low-level Aufgaben.
  • Nützlich für Flags, Masken, Ports, Mikrocontroller.

Was sind Bitoperationen?

Statt auf Ebene ganzer Datentypen arbeiten wir direkt mit Bits.

1000101110100101

Einsatzgebiet

  • Hardwarenahe Programmierung: einzelne Datenleitungen (Bits) steuern
  • Flags in einem Datenwort speichern statt vieler Booleans

Darstellung ganzer Zahlen

In Java hat ein int 32 Bits, long 64 Bits.

int:  0000 0000 0000 0000 0000 0000 0000 0000
long:  0000 ... (64 bits)

Zweierkomplement – positive Zahlen

Höchstes Bit = 0 für positive Zahlen.

Beispiel (32-bit):
0101 1110 0111 0000 0111 0000 0100 0110

Maximale positive Zahl bei n Stellen: 2^(n-1)-1

Zweierkomplement – negative Zahlen

Höchstes Bit = 1 für negative Zahlen. Kleinste darstellbare Zahl: −2^(n-1)

Beispiel (32-bit):
1101 1110 0111 0000 0111 0000 0100 0110

Negieren (Zweierkomplement erzeugen)

  1. Alle Bits invertieren
  2. 1 addieren
4 als Binär:
0000 ...0100
invertieren:
1111 ...1011
+1:
1111 ...1100

Binär → Hex

1111 0000 1100 1001 1111 1111 1111 0110
-> 0xF0C9FFF6

Praktische Beispiele in Java


int flags = 0;
// Bit 2 setzen
flags |= 1 << 2;
// Prüfen ob Bit 2 gesetzt ist
boolean isSet = (flags & (1 << 2)) != 0;
// Bit 2 löschen
flags &= ~(1 << 2);

Weitere Beispiele


// Maskieren
int value = 0xF0C9FFF6;
int masked = value & 0xFF; // unterstes Byte

// XOR zum Umschalten von Bits
value ^= (1 << 5);
