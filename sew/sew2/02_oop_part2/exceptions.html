Exceptions in Java

Fehlerbehandlung einfach erklärt

für Schüler*innen mit Grundkenntnissen in Java

Warum kümmern wir uns um Fehler?

  • Programme laufen nicht immer wie geplant.
  • Ohne Behandlung stürzt das Programm ab.
  • Mit Exceptions kann man Fehler erkennen, melden und sinnvoll reagieren.

Was ist eine Exception?

  • Ein Objekt, das einen Fehler oder ungewöhnlichen Zustand beschreibt.
  • Wird geworfen (throw) und kann gefangen (catch) werden.
  • Beispiele: NullPointerException, ArithmeticException, IOException.

Fehlerarten kurz

  • Compile‑Errors: Code schreibt Java nicht korrekt — Programm startet gar nicht.
  • Runtime‑Errors / Exceptions: treten während der Ausführung auf.
  • Logical Errors: falsches Ergebnis, aber kein Fehler abgestoßen.

try / catch: Grundstruktur

try {
    // Code, der Fehler verursachen könnte
} catch (ExceptionTyp name) {
    // Behandlung des Fehlers
}

Beispiel: Division durch Null

int a = 10;
int b = 0;
try {
    int c = a / b; // ArithmeticException
    System.out.println(c);
} catch (ArithmeticException ex) {
    System.out.println("Fehler: Division durch Null!");
}

finally

  • Wird immer ausgeführt, egal ob eine Exception geworfen wurde.
  • Gut für Aufräumarbeiten, z.B. Streams schließen.
try {
    // ...
} catch (Exception e) {
    // ...
} finally {
    // immer ausführen
}

checked vs. unchecked Exceptions

  • Checked: müssen zur Compilezeit behandelt oder weitergereicht werden (z.B. IOException).
  • Unchecked: RuntimeException und Unterklassen (z.B. NullPointerException) — Behandlung optional.

throw und throws

  • throw new ExceptionType(...): selbst eine Exception auslösen.
  • throws ExceptionType in Methodensignatur: Methode gibt an, dass sie Exceptions weiterreicht.
void readFile(String path) throws IOException {
    if (path == null) throw new IllegalArgumentException("Pfad fehlt");
    // ... kann IOException werfen ...
}

Eigene Exception erstellen

public class InvalidAgeException extends Exception {
    public InvalidAgeException(String msg) {
        super(msg);
    }
}

Verwenden:

if (age < 0) throw new InvalidAgeException("Alter ungültig");

Best Practices

  • Fange nur Exceptions, die du sinnvoll behandeln kannst.
  • Keine leeren catch-Blöcke (keine silent failures).
  • Gib informative Fehlermeldungen / Logging aus.
  • Räume Ressourcen immer auf (try-with-resources oder finally).

try-with-resources

Automatisches Schließen von Ressourcen (seit Java 7).

try (BufferedReader r = new BufferedReader(new FileReader("file.txt"))) {
    String line = r.readLine();
} catch (IOException e) {
    e.printStackTrace();
}

Debugging von Exceptions

  • Stacktrace lesen: Zeigt Klasse, Methode und Zeilennummer.
  • Mit kleinen Tests Ursache eingrenzen.
  • IDE-Breakpoints bei Exceptions setzen.

Kurze Übung (für die Klasse)

  1. Schreibe eine Methode, die zwei Zahlen einliest und teilt.
  2. Fange eine mögliche Division durch Null und gib eine freundliche Meldung aus.
  3. Erweitere: lese Zahlen aus einer Datei, benutze try-with-resources und behandle IOException.

Zusammenfassung

  • Exceptions machen Programme robuster.
  • try/catch/finally sind die Werkzeuge zur Fehlerbehandlung.
  • Checked vs. unchecked bestimmen, wann man verpflichtend behandelt.
  • Gute Fehlerbehandlung hilft beim Testen und beim Anwenderverständnis.
